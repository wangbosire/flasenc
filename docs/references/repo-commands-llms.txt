# flasenc 仓库命令（Docker 标准开发环境 + pnpm 9 + turbo）

在仓库根目录执行。包名见各应用 `package.json` 的 `name` 字段。

## Docker（开发标准路径）

pnpm docker:dev       # 构建并前台启动完整开发栈；应用容器名 flasenc
pnpm docker:up        # 后台启动完整开发栈；MySQL 容器名 flasenc-mysql
pnpm docker:logs      # 查看 flasenc 容器日志
pnpm docker:shell     # 进入 flasenc 容器后执行 pnpm lint/check-types/build/test
pnpm docker:down      # 停止容器，保留数据卷
docker compose down -v # 清空 Docker MySQL 与依赖缓存卷（谨慎）

## 根目录（全仓 / turbo）

pnpm install
pnpm dev              # turbo run dev — 对有 dev 脚本的 workspace 并行（含 @flasenc/admin-web、@flasenc/server）
pnpm build            # turbo run build
pnpm lint             # turbo run lint
pnpm check-types      # turbo run check-types（含 @repo/ui、@flasenc/server、@flasenc/mobile）
pnpm format           # Prettier（根脚本；范围见 package.json）

## 按应用过滤（推荐日常开发）

# 管理后台 React + Vite + shadcn/ui（建议在 `pnpm docker:shell` 内执行）
pnpm --filter @flasenc/admin-web dev
pnpm --filter @flasenc/admin-web build
pnpm --filter @flasenc/admin-web run check-types
pnpm --filter @flasenc/admin-web lint
pnpm --filter @flasenc/admin-web test

# 后端 NestJS（dev 即 nest start --watch；另保留 start:dev 脚本名）
pnpm --filter @flasenc/server dev
pnpm --filter @flasenc/server build
pnpm --filter @flasenc/server lint
pnpm --filter @flasenc/server test
pnpm --filter @flasenc/server run test:e2e
# E2E / 开发数据库默认使用 Docker MySQL（容器 flasenc-mysql，库 flasenc）
pnpm --filter @flasenc/server run check-types
pnpm --filter @flasenc/server run db:test-connection   # MySQL 连通性（见 BACKEND §1.1）
pnpm --filter @flasenc/server run db:migrate:deploy    # 空库 / 新环境应用已版本化迁移
pnpm --filter @flasenc/server run db:seed:admin         # 本地管理账号 admin@example.com / admin123..

# 移动端 uni-app（脚本名带平台后缀；check-types 已接入根 turbo）
pnpm --filter @flasenc/mobile run dev:h5
pnpm --filter @flasenc/mobile run build:h5
pnpm --filter @flasenc/mobile run check-types
pnpm --filter @flasenc/mobile run type-check     # check-types 的兼容别名

## 共享包

pnpm --filter @repo/ui lint
pnpm --filter @repo/ui run check-types

## 说明

- `@flasenc/mobile` 使用 `check-types`（vue-tsc），`type-check` 保留为兼容别名；全仓 `pnpm check-types` 会覆盖 mobile。
- 根 `pnpm dev` 会并行拉起所有带 `dev` 的 workspace（含 admin-web 与 server）；若只需其一，用 `--filter`。
- `@flasenc/server` 的 `format` 脚本当前路径指向仓库内 `apps/**/*.ts` 等，若在 monorepo 根执行需注意工作目录；建议在 `apps/server` 目录下执行或后续改为仅格式化 `apps/server`。
