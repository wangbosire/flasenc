## 1. 背景与目标

平台通过 **内容权益 + 兑换码** 将占位内容的处置权交给 **已注册** C 端用户；用户可编辑、选用 **内容模板（整稿替换）**、走机审后发布。访客可读已发布内容；登录用户可参与 **对话串式二层** 评论；可配置 **评论通知**（站内信、小程序订阅消息等）。**内容 Owner** 可按规则 **转让** 所有权（卡片或转让码，**7 个自然日**至第 7 日 **23:59:59**）。平台可人工队列、下架、隐藏与审计。**机审供应商**待定，验收仍以本文 **§6、§7、§5** 行为为准。

---

## 2. 范围与非目标

**范围内（MVP）**：一权益一内容、占位随权益创建；兑换码一码一用、默认长期有效、可置失效；**须登录**兑换；发布与评论机审分支（通过 / 拦截 / 疑似 + 人工）；模板与内容 **非关联管理**、选用 **免审计**、发布仍机审；评论二层 + 加载更多 + 通知偏好；Owner 转让（撤销/确认/过期、管理端转让记录）；平台干预与 **§13 审计**（豁免见 §13）。

**非目标**：一码多用、一权益多内容；评论 **第三层** 父子；机审选型细节（见 **§11**）。

---

## 3. 角色（摘要）

| 角色 | 要点 |
|------|------|
| 平台人员 | 权益与码、模板 CRUD/下架/删除、疑似队列、手动送审、下架/隐藏、删任意评、按内容查 **转让记录** |
| 内容 Owner | 编辑/发布/选模板/发起转让/删本帖任意评/通知偏好 |
| 前 Owner（已转让） | 失 Owner 权；可读已发布、可发评（同普通用户） |
| 评论作者 / 管理员 / 访客 | 自删；全站删评；访客只读不可兑不可评 |

---

## 4. 规则总表（验收口径）

| 域 | 规则 |
|----|------|
| 权益与码 | 创建权益 **自动创建**绑定 **占位** 内容；一权益一内容；生成码；一码一用、用后作废 |
| 兑换 | **须注册并登录**；未登录调用兑换 **拒绝**（与 API `error.code` 一致） |
| 码生命周期 | 默认 **永久**；平台可 **置失效**；须审计 |
| 访问与评论 | 访客只读已发布；发评须登录；**锚点 + 串内第二层**（`parent` 均指向锚点；`replyToCommentId` 仅展示）；**禁止第三层**；列表 **加载更多** |
| 通知 | 新评/回复触发；用户可勾选 **站内信**、**小程序订阅消息**（他端非目标须写明） |
| 模板 | 与内容无绑定表；**整稿替换**；文字/图/自定义动画；保存无模板审；**选用不写审计**；下架可编辑元数据与正文、可恢复；删除须权限+二次确认、不可恢复；仅 **上架中** 可选 |
| 转让 | Owner 发起；**卡片** 或 **转让码**；发起日算第 1 日，**第 7 日 23:59:59** 截止；仅一笔 **待确认**；可撤销；确认后 Owner 转移；原 Owner 保留读评；**不** 新建权益；管理端 **转让记录** 可查 |
| 平台 | 手动送审、**无条件下架**、**紧急隐藏**；均须审计 |
| 机审（内容） | 通过 → 正常已发布；异常 → 拦截提示；疑似 → 先发布进队列；人工标记失败 → Owner 可再编再提 |
| 机审（评论） | 通过展示；失败拦截；疑似先发布进队列；人工标记失败 → 系统删该条（实现二选一写死） |

---

## 5. 用户故事（精简）

1. 访客可读已发布，不可兑、不可评。  
2. 用户须登录后才可兑换。  
3. 平台创建权益即带出占位内容与码；码可失效。  
4. Owner 编辑、可选用上架模板整稿替换后再改、再提交发布。  
5. 登录用户在已发布内容下按二层规则发评/串内回复；列表加载更多。  
6. 作者删己评；Owner 删本帖任意评；管理员删任意评。  
7. Owner 发起转让（卡/码）；受让人确认后成为新 Owner；原 Owner 失权但可评。  
8. 平台查看某内容的转让时间线。  
9. 审计方可按人/物/动作/时间检索 **§13** 所列（选用模板豁免）。

---

## 6. 验收标准（可勾选，按域）

**权益与兑换**：占位随权益；一码一用并发安全；未登录兑换拒绝且码明确；已登录可兑；失效码不可兑；默认长期有效。

**内容与发布**：§6 三分支 + 人工标记失败可再提；下架/隐藏后访客侧与 **§13** 一致。

**评论**：未登录只读；发评须登录；二层结构校验（含拒绝第三层）；`replyToCommentId` 不破坏父子深度；加载更多；通知与偏好及 **§13（若记）**。

**平台**：疑似队列可处理；手动送审、下架、隐藏有入口且留痕。

**删评**：作者 / 管理员 / Owner 三条路径可验收。

**模板**：新建编辑无模板审；选用整稿替换、无选用审计；下架/删除与不可再选规则；模板变更不破坏已替换正文。

**转让**：双入口一致状态机；7 日规则与时区一致；自转让拒绝；确认前后权限；撤销与过期留痕；单笔 pending；管理端记录只读与权限 **403**。

**审计**：§13 每类至少一条 E2E 验证；管理端可按时间+类型查。

---

## 7. 机审分支（内容 / 评论）

**内容发布**与 **评论** 均区分：**通过**、**审核异常/失败（拦截）**、**疑似（先发布进队）**、**人工标记失败**（内容侧 Owner 可再提；评论侧删除该条）。正文载体（含动画）安全与性能由技术方案定义，验收不降低 §6/§7 行为。

---

## 8. 兑换码生命周期（权益码）

默认永久；平台可置失效（须审计）；已使用与失效独立。**转让码** 不适用「永久默认」，适用 **§2、§4** 中的 **7 个自然日** 转让规则。

---

## 9. 平台干预（摘要）

手动送审；无条件下架；紧急隐藏。与 Owner 权并存时 **平台合规优先**；须 **§13**。

---

## 10. 评论形态（二层）与删评

- **结构**：多条并行串；每串 **锚点（层 1）** + **仅挂在锚点下的串内消息（层 2）**；回复某条二层消息时 **新节点仍为锚点子节点**，可用 `replyToCommentId` 做展示，**禁止** `parent` 指向非锚点而形成深度 ≥3。  
- **删评**：见 **§3**；删除须 **§13**。

## 13. 审计留痕

**原则**：下列操作须落 **审计日志**（谁、何时、对象、动作、可选摘要）；业务用户不可篡改；保留与脱敏见 [`../SECURITY.md`](../SECURITY.md)。

**豁免（产品明确）**：**Owner 选用内容模板（整稿替换）** 不要求审计。

**须覆盖的类别（示例）**：注册/登录/失败；评论偏好（若记）；权益与码（创建、生成、失效等）；模板创建/更新/下架/恢复/删除（不含选用）；兑换成功/失败；转让发起/撤销/确认/过期（及定时任务）；内容草稿/提交/机审/人工/送审/下架/隐藏/权属变更；评论发表/机审/人工/各角色删除；队列处理（若需要）。
