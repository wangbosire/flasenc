# Harness 核心信念

- **人类掌舵，智能体执行**：范围、风险、发布与关键验收由人拍板；智能体在清晰工件与 CI 下辅助实现与文档。  
- **仓库即记录系统**：结论进 `docs/`、Issue、PR；未进库则对后续协作者与智能体视为不存在。  
- **给地图、不给百科全书**：`AGENTS.md` 与索引保持短小；流程与细节分层在 `development-lifecycle.md` 与专题文档。  
- **反馈回路**：自测、CI、QA、监控均为改进输入；重复问题上升为检查项或自动化。  
- **小步可回滚**：与团队门禁一致的前提下，优先可验证、可合并的小变更。
